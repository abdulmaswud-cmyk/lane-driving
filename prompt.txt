### 1. App Overview

Build a **native mobile car game** using **Expo + React Native** where the player controls a car that **switches lanes to avoid obstacles**.

The app should be:

* Mobile-first (iOS & Android)
* Built with Expo
* Clean, responsive, and performant
* Deployable via **Vercel (Expo Web build)**

### 2. Technical Requirements

* **Framework:** Expo (React Native)
* **Language:** TypeScript
* **State Management:** React hooks (useState, useEffect, useRef)
* **Game Loop:** requestAnimationFrame or setInterval
* **Navigation:** expo-router or React Navigation
* **API Fetching:** fetch / axios
* **Animations:** Animated API or Reanimated
* **Deployment Target:** Vercel (Expo Web)

### 3. Core Game Mechanics

* Vertical scrolling road with **3 lanes**
* Player car can switch lanes via:

  * Swipe left / right
  * Or tap buttons
* Obstacles spawn randomly and move downward
* Collision detection ends the game
* Increasing difficulty over time (speed & spawn rate)

### 4. Features

#### Gameplay

* Lane switching
* Score counter (distance survived)
* Increasing difficulty
* Game over & restart screen

#### Recipe API Integration

* Fetch recipes from an **open public recipe API**
* Recipes appear as:

  * Power-ups (speed boost, shield, slow obstacles)
  * Visual collectibles
  * Score multipliers
* Display recipe name + thumbnail when collected

#### UI

* Start screen
* Game screen
* Game over screen
* Optional recipe detail modal

### 5. API Specification

Use a **free and open recipe API**, such as:

**TheMealDB (recommended)**

* Base URL: `https://www.themealdb.com/api/json/v1/1`
* Example endpoint:
  `/search.php?s=chicken`

**Usage in Game**

* Fetch random recipes on game start
* Store recipe list in state
* Randomly spawn recipe items on the road
* On collection, apply effect based on recipe category or area
